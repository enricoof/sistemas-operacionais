Foco na Prova - Linux, Bash e Gestão de Processos

Questões Quentes para a Prova

1. Qual a diferença entre Software Livre e Software Gratuito (Freeware)?

O fato de ser gratuito não implica que o software seja livre[cite: 61]. O software gratuito (freeware) apenas não cobra pela licença de uso, mas seu código-fonte permanece fechado e indisponível[cite: 62]. Já o Software Livre garante ao usuário as liberdades fundamentais de executar, copiar, estudar, modificar e redistribuir o código-fonte, independentemente de ser comercializado ou não[cite: 48, 54, 55, 56].

2. Defina o que é um Sistema Operacional (SO) e suas principais funções.

O Sistema Operacional é um programa que atua como intermediário entre o usuário e o hardware do computador[cite: 3794]. Ele opera em duas frentes principais: como um gerenciador de recursos (alocando ordenadamente CPU, memória e I/O para os programas) e como uma máquina virtual ou estendida, oferecendo abstrações que escondem os detalhes complexos do hardware e facilitam o uso[cite: 12, 14, 15].

3. Explique a diferença entre Multiprogramação e Time-Sharing (Tempo Compartilhado).

A Multiprogramação permite que diversos processos sejam mantidos na memória simultaneamente; quando um processo aguarda uma operação de I/O, a CPU é alocada para outro processo, maximizando seu uso[cite: 4108, 4132]. Já o Time-Sharing é uma evolução lógica voltada à interatividade: o processador é alocado por uma fatia de tempo (time slice) para cada usuário/programa em modo circular, dando a ilusão de que a máquina é dedicada a cada usuário simultaneamente[cite: 3575, 3577].

4. O que é o algoritmo Round Robin (RR) e o que é o Quantum?

O Round Robin é um algoritmo de escalonamento preemptivo ideal para sistemas de tempo compartilhado. Ele atribui a cada processo uma pequena e igual unidade de tempo da CPU, chamada Quantum[cite: 5189, 5190]. Quando o quantum de um processo expira, o sistema operacional o interrompe (preempção) e o coloca no final da fila de processos prontos, passando a CPU para o próximo processo da fila[cite: 5191].

5. O que é Starvation (Inanição) e qual técnica resolve esse problema?

Starvation é um fenômeno onde um processo aguarda indefinidamente pela alocação da CPU porque outros processos (geralmente de maior prioridade) continuam chegando e assumindo a execução[cite: 4542, 4545]. A solução padrão em SOs é o Aging (Envelhecimento), técnica que aumenta gradualmente a prioridade dos processos que estão aguardando há muito tempo na fila de prontos[cite: 4976].

6. Diferencie o funcionamento dos pipes ( | ) e dos redirecionadores ( > e >> ) no Bash.

Os redirecionadores manipulam a entrada e saída para arquivos. O > redireciona a saída de um comando sobrescrevendo o arquivo de destino, enquanto o >> anexa a saída ao final do arquivo[cite: 846, 847, 849]. O pipe ( | ) cria um encadeamento entre comandos: ele pega a saída (standard output) do comando executado à esquerda e a injeta como entrada (standard input) para o comando à direita[cite: 1830, 5533, 5535].

Resumo Rápido (Cheat Sheet)

Comandos Essenciais de Terminal

  • pwd: Print Working Directory (Mostra onde você está)[cite: 1055].
  • cd /caminho: Change Directory (Navega entre pastas). cd .. sobe um nível[cite: 1060, 1064].
  • ls -la: Lista arquivos, incluindo os ocultos e com permissões[cite: 951].
  • mkdir / rmdir: Cria / Remove diretório (rmdir só apaga se estiver vazio)[cite: 898, 925].
  • rm -rf: Remove diretórios não vazios recursivamente e sem pedir confirmação[cite: 1183, 1189].
  • cp -r: Copia diretórios de forma recursiva[cite: 1202].
  • mv: Move OU renomeia arquivos/diretórios[cite: 1093, 1095].
  • cat: Concatena e imprime o conteúdo de um arquivo na tela[cite: 792].
  • grep: Busca por strings/padrões em arquivos (ex: grep -i "texto" para ignorar maiúsculas)[cite: 1311, 1316].
  • chmod: Altera permissões. chown: Altera o dono do arquivo.

Gestão de Processos

  • Processo: Programa em execução[cite: 2046]. Tem um PID (Identificador) único[cite: 2050].
  • Foreground vs Background: Foreground toma o terminal[cite: 2074]. Background (use o & no final do comando) roda em paralelo[cite: 2086, 2091].
  • Suspensão: Ctrl + Z pausa um processo e o joga pro background[cite: 2101]. Use bg para retomá-lo atrás das cortinas, ou fg para trazê-lo de volta[cite: 2108, 2112].
  • Mata Processo: kill PID encerra[cite: 2170]. kill -9 PID força a morte imediata (SIGKILL)[cite: 2500, 2560].

Algoritmos de Escalonamento (O que Cai!)

  • FCFS (First-Come, First-Served): Primeiro a chegar é o primeiro a usar a CPU. Não preemptivo. Pode gerar alto tempo de espera[cite: 4775, 4778].
  • SJF (Shortest Job First): Executa o processo mais curto. É ótimo para o tempo médio, mas não dá pra saber o tempo futuro de um processo na vida real[cite: 4837, 4845].
  • SRT (Shortest Remaining Time): Versão preemptiva do SJF[cite: 4907].
  • RR (Round Robin): Usa fatias de tempo (quantum). Se o processo não acabar, sofre preempção. Justo e bom para interatividade[cite: 5189, 5190].
  • Prioridade: Processos com maior prioridade passam na frente (números menores geralmente indicam prioridade maior)[cite: 4971, 4992].

Conteúdo Aprofundado

1. A Filosofia GNU e Linux

O Projeto GNU foi fundado por Richard Stallman em 1984 com o objetivo de criar um sistema operacional 100% livre (GNU's Not Unix)[cite: 36, 37, 39]. A liberdade de software é garantida pela licença GPL e abrange as quatro liberdades: executar, estudar, redistribuir e modificar/aperfeiçoar o código[cite: 46, 54, 55, 56, 57, 58].

O Linux, criado por Linus Torvalds em 1991, é apenas o Kernel (núcleo) do sistema[cite: 132, 216]. Quando juntamos o kernel Linux com os utilitários GNU, formamos o sistema operacional completo que usamos hoje[cite: 224]. O Tux (pinguim) é a mascote oficial[cite: 171].

2. Estrutura do SO e Gerência de Processos

O PCB (Process Control Block) é uma estrutura que guarda o contexto de um processo (seu PID, registradores, program counter, estado atual) para que a CPU saiba exatamente de onde retomar o processamento após uma preempção[cite: 4491, 4492, 4510].

Ciclo de vida do processo:

  • New: Sendo criado[cite: 2354].
  • Ready (Pronto): Aguardando a CPU na fila do escalonador[cite: 2363].
  • Running (Executando): De posse do processador[cite: 2366].
  • Waiting (Bloqueado): Pausado aguardando um recurso ou um I/O[cite: 2374].
  • Terminated: Concluído[cite: 2355].

3. A Profundidade do Escalonamento

O escalonador de curto prazo atua em milissegundos selecionando quem ganha a CPU[cite: 4581, 4582]. Ele avalia o ciclo do processo que alterna entre uso intenso da CPU (CPU Bound) e espera por entrada/saída (I/O Bound)[cite: 4631, 4634].

O sistema conta com o processo idle para consumir ciclos de CPU quando todas as outras tarefas estão bloqueadas. Ele roda com a menor prioridade possível para nunca atrapalhar as tarefas úteis e evita a ociosidade total abrupta da máquina[cite: 5353, 5354, 5355].

Flashcards Interativos (Clique para virar)

Revise os conceitos de memorização rápida.

O que o comando 'chmod' faz?

Altera as permissões de acesso (leitura, escrita, execução) de arquivos e diretórios.

O que é Preempção?

É a capacidade do Sistema Operacional (Escalonador) de interromper e retirar a CPU de um processo à força, antes dele terminar[cite: 4558].

Diferencie Caminho Absoluto e Relativo.

Absoluto: Parte sempre do diretório raiz ('/'). Ex: /etc/hosts[cite: 855, 856].

Relativo: Parte do diretório em que você se encontra. Ex: marcio/arq.txt[cite: 858, 859].

Qual a finalidade do comando 'nohup'?

Manter um processo rodando no sistema mesmo após o usuário fazer logout ou fechar o shell[cite: 2873, 2874].

O que faz o algoritmo SJF?

Shortest Job First. Ele aloca a CPU para o processo que necessita do menor tempo de execução[cite: 4837, 4840]. É ótimo para a média de tempo de espera[cite: 4845].

O que é o Spooling?

Técnica onde as operações de I/O são transferidas para buffers (como discos) para que a CPU continue processando outras tarefas simultaneamente, sem ociosidade[cite: 4070, 4071].
