Conceitos Teóricos Fundamentais

1. O que é um Sistema Operacional? [cite: 2957, 2979, 3204]

O Sistema Operacional (SO) é o software fundamental que atua como intermediário entre o hardware do computador e os aplicativos do usuário. Ele possui duas grandes responsabilidades clássicas:

  • Máquina Estendida (ou Virtual): O hardware nu é complexo e difícil de programar. O SO cria uma "ilusão" de uma máquina muito mais simples, limpa e padronizada (oferecendo abstrações como janelas, arquivos e processos) para que o usuário e o programador não precisem lidar com detalhes físicos do disco ou do processador.
  • Gerenciador de Recursos: Quando há vários usuários ou programas rodando, o SO gerencia a ordem e o tempo de uso da CPU, a alocação de memória RAM e o acesso aos dispositivos de Entrada/Saída (I/O), garantindo proteção e concorrência ordenada.

2. Arquiteturas e Tipos de SO [cite: 3491, 3507, 3574, 4304, 4309, 4345]

Os SOs evoluíram para lidar com necessidades específicas de processamento:

  • Batch (Lote): O usuário não interage com o programa em execução. Os programas são processados em fila (sequencialmente), otimizando a máquina, mas zerando a interatividade.
  • Multiprogramação: Mantém vários processos na memória ao mesmo tempo. Se o processo "A" pausa para ler o disco (I/O), o SO joga o processo "B" na CPU. Maximiza o uso do hardware.
  • Time-Sharing (Tempo Compartilhado): É uma evolução da multiprogramação focada na interatividade. A CPU é dividida em pequenas "fatias de tempo" (time slices). Alterna tão rápido entre os processos que cada usuário acha que a máquina é exclusiva dele.
  • Multiprocessamento: Uso de múltiplas CPUs. Pode ser Assimétrico (uma CPU mestre dita as regras e as outras obedecem) ou Simétrico / SMP (todas as CPUs são iguais e compartilham a mesma memória).
  • Sistemas Distribuídos: Várias máquinas (nós) independentes conectadas em rede, cada uma com sua memória local, que parecem um sistema único para o usuário.
  • Sistemas de Tempo Real (Real-Time): A exatidão temporal é crítica. Subdividido em Hard (se perder o prazo/deadline, ocorre uma falha catastrófica - ex: robótica médica, aviões) e Soft (se perder o prazo há perda de qualidade, mas não é fatal - ex: streaming de vídeo). O Windows padrão NÃO é de tempo real.

3. Gestão de Processos e Context Switch [cite: 4492, 4557, 4688, 4708]

Um processo é um "programa em execução". O kernel gerencia os processos utilizando o PCB (Process Control Block ou Bloco de Controle de Processo), que é uma estrutura de dados que guarda o "estado atual" do processo (onde o Program Counter parou, quais registradores estavam em uso, limites de memória, etc.).

Mudança de Contexto (Context Switch): Quando a CPU precisa sair do Processo A e ir para o Processo B, ela precisa salvar o estado do Processo A no PCB dele, e carregar o estado do Processo B do PCB dele. Isso é custoso e gera latência (Dispatch Latency).

Preempção: É o ato do SO interromper forçadamente um processo que está rodando na CPU para dar lugar a outro (seja porque o tempo dele acabou, ou chegou alguém mais prioritário).

4. Escalonamento da CPU: Algoritmos [cite: 4775, 4839, 4907, 4975, 5190, 5421]

O Escalonador de Curto Prazo (Short-Term Scheduler) escolhe qual processo da "Fila de Prontos" vai para a CPU. A execução de um processo é um ciclo contínuo de CPU Burst (uso do processador) e I/O Burst (espera por periféricos). Os algoritmos decidem a ordem:

• FCFS (First-Come, First-Served)

Não preemptivo. O primeiro que chega é o primeiro a usar a CPU até terminar.
Problema: Se um processo enorme chegar primeiro, todos os menores vão ficar travados atrás dele (Efeito Comboio), gerando alto tempo de espera médio.

• SJF (Shortest Job First)

Não preemptivo. Pega o processo com o menor tempo de CPU (CPU Burst) da fila e o executa.
Vantagem: É matematicamente o algoritmo que gera o menor tempo médio de espera.
Problema: Na prática, o SO não tem uma bola de cristal para prever o exato tempo futuro que um processo vai gastar na CPU.

• SRT (Shortest Remaining Time)

É a versão preemptiva do SJF. Se um processo novo chegar com um tempo de CPU menor do que o tempo restante do processo atual, o atual sofre preempção e perde a CPU para o novato.

• Escalonamento por Prioridade & Starvation

Preemptivo ou não. Cada processo ganha um número de prioridade (geralmente, número menor = prioridade maior). A CPU vai para a maior prioridade.
O grande problema: Starvation (Inanição). Processos de baixa prioridade podem ficar travados para sempre se processos de alta prioridade não pararem de chegar.
A solução: Aging (Envelhecimento), que aumenta gradualmente a prioridade dos processos que estão aguardando há muito tempo.

• Round Robin (RR)

Preemptivo. Projetado para Time-Sharing. Cada processo ganha uma fatia de tempo exata chamada Quantum (ex: 20ms). Se o processo não terminar dentro do Quantum, ele sofre preempção e vai pro fim da fila. É o mais justo para interatividade. Se o Quantum for muito grande, vira um FCFS. Se for muito pequeno, a máquina perde muito tempo apenas fazendo mudança de contexto.

• Múltiplas Filas (Multilevel Queues)

Divide os processos em várias filas com prioridades distintas (Ex: Fila Foreground usa Round Robin; Fila Background usa FCFS). Em Múltiplas Filas com Feedback, os processos podem transitar entre as filas de acordo com seu comportamento e consumo de CPU.

Navegação e Sistema de Arquivos

pwd

Print Working Directory: Mostra o caminho absoluto do diretório onde você está no momento.

cd [diretório]

Change Directory: Muda a pasta atual. cd .. sobe um nível. cd / vai para o diretório raiz. Digitar só cd volta para o diretório Home (~) do usuário.

ls [opções]

List: Lista arquivos e diretórios.
ls -l: lista longa (mostra permissões, dono, tamanho, data).
ls -a: mostra arquivos ocultos (que começam com ponto).
ls -lh: mostra o tamanho em formato legível para humanos (MB, GB).

mkdir [nome] / rmdir [nome]

Make / Remove Directory: Cria ou remove pastas. O rmdir só apaga a pasta se ela estiver completamente vazia.

cp [origem] [destino]

Copy: Copia arquivos. Para copiar pastas e todo seu conteúdo, use cp -r (recursivo).

mv [origem] [destino]

Move: Move arquivos de um lugar para outro, OU renomeia um arquivo, dependendo do destino informado.

rm [arquivo]

Remove: Apaga arquivos.
rm -r: apaga pastas recursivamente.
rm -f: força a deleção sem perguntar (cuidado com rm -rf).

touch [arquivo]

Cria um arquivo vazio. Se o arquivo já existir, ele apenas atualiza a data e hora de acesso/modificação do arquivo.

Manipulação e Visualização de Texto

cat [arquivo]

Concatenate: Imprime todo o conteúdo de um arquivo de texto no terminal de uma só vez.

more / less

Mostra o conteúdo de um arquivo grande página por página. O less é mais flexível, permitindo navegar para cima e para baixo.

grep [padrão] [arquivo]

Busca por palavras ou padrões de texto dentro de arquivos.
grep -i: ignora diferença entre maiúsculas e minúsculas.
grep -v: inverte a busca (mostra linhas que NÃO contêm a palavra).

Redirecionamentos e Pipes

> e >>

> (Sobrescreve): Pega a saída de um comando e salva em um arquivo, apagando o que tinha lá.
>> (Adiciona): Salva a saída no final do arquivo, preservando o que já existia (append).

| (Pipe)

Pega a saída do comando da esquerda e joga diretamente na entrada do comando da direita (Ex: ls -l | more).

Gerenciamento de Processos

Comando &

Colocar um & no final de qualquer comando faz com que ele execute em Background (segundo plano), liberando o terminal para você continuar usando.

Ctrl+Z / bg / fg / jobs

Ctrl+Z: Suspende/pausa o processo atual.
jobs: Lista os processos rodando no background.
bg %1: Retoma a execução do processo 1 em background.
fg %1: Traz o processo 1 de volta para o primeiro plano (Foreground).

ps [opções]

Process Status: Mostra os processos atuais.
ps -e (ou -A): mostra processos de todos os usuários.
ps -f: saída completa (full). Ex: ps -ef é muito comum.

top / htop

Monitores de processos dinâmicos. Ficam atualizando a tela em tempo real, mostrando quem consome mais CPU e RAM. O htop é a versão colorida e melhorada.

kill / killall

Envia sinais (geralmente para finalizar processos).
kill PID: encerra um processo suavemente (SIGTERM -15).
kill -9 PID: mata forçadamente, sem chance do processo salvar dados (SIGKILL).
killall [nome]: mata todos os processos que tenham o mesmo nome.

nice / renice

Altera a prioridade de um processo. O valor de prioridade (NI) vai de -20 (prioridade altíssima) a +19 (prioridade baixíssima). O padrão é 0.
nice: define a prioridade ao iniciar.
renice: altera a prioridade de um processo já rodando.

nohup

No Hang Up: Garante que o processo continue rodando no background mesmo que você feche a janela do terminal ou faça logout.

Outros Úteis

df -h (Vê espaço em disco em MB/GB), uptime (Há quanto tempo o PC está ligado e a carga média), whoami (Mostra seu usuário logado), clear (Limpa a tela do terminal).

Flashcards de Fixação (Toque para Virar)

As questões clássicas de prova baseadas nos PDFs.

O que é a "Inversão de Prioridade" e como a "Inanição (Starvation)" é resolvida?

Inversão ocorre quando um processo de baixa prioridade trava um de alta. A Inanição ocorre no escalonamento por prioridade e é resolvida pelo Aging (Envelhecimento): aumentar gradualmente a prioridade dos processos que aguardam há muito tempo.

Qual a diferença entre os escalonadores SJF e SRT?

Ambos escolhem o processo mais curto. O SJF é Não-Preemptivo (deixa terminar). O SRT (Shortest Remaining Time) é Preemptivo: se chegar um menor, ele rouba a CPU do atual.

Explique a função do Quantum no algoritmo Round Robin.

Quantum (ou fatia de tempo) é o limite máximo de tempo contínuo que um processo pode usar a CPU por vez. Quando o quantum acaba, o processo sofre preempção e vai pro final da fila.

O que acontece se o SO não tem NENHUM processo pronto para rodar?

O Escalonador aciona o processo Idle (Processo Ocioso). Ele tem a menor prioridade possível e roda apenas para evitar que a CPU gaste tempo travando/reiniciando ou fazendo busy wait inútil.

Como jogar um processo para Background e Background pausado no Linux?

Use & no final do comando para iniciar direto em background. Se já estiver rodando, aperte Ctrl+Z para suspender e depois digite bg para ele rodar escondido.

Para que servem os comandos nice e kill -9?

nice ajusta a prioridade do processo (valores negativos = maior prioridade). kill -9 manda o sinal SIGKILL, que força a morte imediata do processo, sem chance de salvar nada.
