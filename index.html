Catálogo Completo de Comandos (A-Z) e Teoria de Sincronização

Guia Definitivo de Comandos (Linux)

Lista exaustiva extraída de todos os módulos de laboratório.

1. Navegação e Arquivos Básicos

pwd

Print Working Directory. Mostra o caminho da pasta atual.

cd [diretório]

Change Directory. Muda de pasta. `cd` sem argumentos volta para o home.

ls [opções]

Lista arquivos. -l (lista longa detalhada), -ld (detalhes apenas do diretório, sem listar conteúdo).

mkdir [pasta]

Make Directory. Cria diretórios.

rmdir [pasta]

Remove Directory. Remove diretórios, somente se estiverem vazios.

cp [opções] [origem] [destino]

Copy. Copia arquivos. -r para copiar diretórios recursivamente.

mv [origem] [destino]

Move. Move arquivos/diretórios de local, ou os renomeia.

rm [opções] [arquivo]

Remove. Apaga arquivos. -r para apagar pastas e seus conteúdos recursivamente.

touch [arquivo]

Cria um arquivo vazio ou atualiza as timestamps (acesso/modificação) de um já existente.

file [arquivo]

Testa e determina o TIPO de um arquivo lendo seus Magic Numbers (cabeçalho).

alias

Lista os apelidos (atalhos) configurados no Shell BASH.

2. Buscas, Filtros e Informações do Sistema

find [onde] [opções]
Pesquisa avançada recursiva (não usa índice pré-construído). Opções principais:
  • -name: Busca por nome (ex: `find / -name "*.txt"`).
  • -size +x / -x: Busca por tamanho maior (+) ou menor (-) que x.
  • -type x: Por tipo (f: arquivo, d: diretório, l: link).
  • -user / -group: De um usuário ou grupo específico.
  • -mmin / -mtime: Modificados há X minutos/dias.
localize (ou slocalize) [nome]

Busca hiper-rápida pois usa um banco de dados pré-indexado.

which [comando]

Encontra o caminho absoluto do executável (baseado na variável PATH).

grep [padrão] [arquivo]

Filtra texto. ^padrão (linhas que começam com o padrão).

cut -d [delimitador] -f [coluna]

Corta colunas de texto. Ex: `cut -d: -f1` usa ":" como separador e pega a coluna 1.

env / echo $PATH / export

`env` exibe variáveis de ambiente. `echo $PATH` mostra caminhos de executáveis. `export` adiciona novos paths.

fdisk [disco]

Exibe ou manipula a Tabela de Partições (ex: `fdisk /dev/sda`).

df [opções]

Disk Free. Vê espaço livre do HD montado. -h (human-readable), -k (em Kb), -T (mostra tipo do file system).

uptime [opções]

Mostra há quanto tempo o SO está ligado, usuários e Carga Média. -p (estilo contagem), -s (estilo data).

3. Gerenciamento de Processos (Foreground/Background)

[comando] &

Joga o processo para o Background.

Ctrl + Z / Ctrl + C

`Ctrl+Z` Suspende/pausa o processo (coloca em background). `Ctrl+C` mata o processo atual no foreground.

bg [%N] / fg [%N]

`bg` retoma execução de um job pausado no background. `fg` traz pro foreground.

jobs

Lista os jobs atuais em background do terminal (+ indica o padrão).

ps [opções]
Exibe status de processos.
  • -e / -A: Todos os processos do sistema.
  • -a: Todos (menos do console).
  • -f: Saída completa (árvore). Famoso `ps -ef`.
  • -u: Inclui nome de usuário e hora.
  • -T: Lista as Threads do processo.
  • -p [PID]: Busca só aquele PID.
pstree [-p]

Mostra os processos em forma de árvore. -p mostra o PID junto.

top [opções] / htop
Monitores em tempo real. Atalhos essenciais no top:
  • c: Mostrar caminho absoluto.
  • d: Alterar tempo de atualização (refresh).
  • k: Matar processo.
  • r: Alterar o nice (prioridade).
  • P (Maiúsculo): Ordenar por %CPU.
  • m: Ordenar por %Memória.
  • Shift+H: Mostrar Threads ao invés de Processos.
pidof [-s] [nome]

Devolve os PIDs daquele programa. -s devolve apenas um único PID.

kill [sinal] [PID] / killall
Envia sinal ao processo.
  • Sem argumento: Padrao -15 (SIGTERM).
  • -9 (SIGKILL): Morte forçada imediata (não salva nada).
  • -STOP / -CONT: Pausa e Continua o processo.
  • killall [-u user] [nome]: Mata todos processos de um nome específico ou de um usuário (-u).
nice [-n] / renice [-n] [-u]

`nice` inicia processo com prioridade alterada (vai de -20 a +19). `renice` altera o de um já rodando (ex: `renice 10 [PID]` ou `renice 20 -u joao`).

nohup [comando] &

No Hang Up. O processo sobrevive ao logout do terminal.

sleep [tempo]

Pausa execução do bash (2s, 2m, 2h).

time [comando]

Retorna estatísticas de tempo real, de user mode e de kernel (sys) após o comando rodar.

strace [-tt, -c, -T] [comando]

Analisa chamadas de sistema (System Calls). -c coleta contagens, -T vê o tempo gasto na chamada.

4. Gerenciamento de Usuários e Grupos

whoami / id

`whoami` mostra seu login. `id` mostra seu UID, GID e grupos adicionais.

adduser [usuário] / useradd [usuário]

Cria usuário. (Adduser é um script mais amigável, useradd é o comando base).

userdel [-r] [usuário]

Apaga conta. -r apaga também o diretório /home/usuario.

usermod [opções] [usuário]
Modifica as propriedades da conta:
  • -l: Altera o nome de login (rename).
  • -aG: Adiciona a um grupo extra sem remover dos atuais.
  • -G: Substitui todos os grupos extra.
  • -d (-m): Altera o diretório home (e -m move o conteúdo).
  • -s: Altera o shell padrão (ex: /bin/bash).
  • -L / -U: Lock (Tranca) e Unlock (Destranca) a conta.
  • -e: Define data de expiração.
passwd [usuário]
Altera senhas criptografadas (vai para /etc/shadow).
  • -l / -u: Lock (tranca senha) e Unlock.
  • -d: Delete (deixa sem senha).
  • -e: Expire (força trocar no próximo login).
  • -S: Status da senha.
chfn [opções] [usuário]

Change finger. Altera dados informativos: -f (nome completo), -o (sala), -p (tel trab), -h (tel casa).

groupadd / groupmod / groupdel

Gerencia grupos. groupmod possui opções -g (novo GID), -p (senha), -a/-R (add/remove usuários).

groups [usuário]

Mostra os grupos do qual um usuário faz parte.

getent [database]

Busca nos dados do sistema (ex: `getent group` ou `getent passwd`). Ignora se vem de arquivo local ou LDAP.

pwconv

Password Convert. Migra/atualiza as senhas do `/etc/passwd` e cria o mascaramento com "x" jogando a criptografia protegida pro `/etc/shadow`.

5. Permissões e Links (Inodes)

chmod [opções] [permissões] [arquivo]
Change Mode. -R altera pastas recursivamente.
Modo Numérico: (R=4, W=2, X=1). Ex: `chmod 764 file`.
Modo Simbólico: u (user), g (group), o (other), a (all) com +, - ou =. Ex: `chmod g+x file`.
chown [dono]:[grupo] [arquivo]

Altera o Owner do arquivo.

chgrp [-R] [grupo] [arquivo]

Altera apenas o Group Owner do arquivo.

umask [-S] [valor]

Mascara (SUBTRAI) permissões padrão ao criar novo arquivo. Ex: `umask 022` tira W de grupo/outros. -S mostra resultado de forma simbólica.

ln [-s] [alvo] [link]
Cria links (ponteiros).
  • Sem -s: Hard Link (aponta direto pro Inode). Não cruza partição. Se apagar original, o arquivo vive pelo link.
  • Com -s: Symbolic Link (Symlink). Aponta pro caminho (texto). Se o original sumir, fica "quebrado".
unlink [link]

Remove (deleta) links simbólicos de forma segura.

Revisão Teórica (Concorrência, Threads e Permissões)

Threads (Processos Leves)

Threads são fluxos de execução dentro de um mesmo processo. Criar/destruir threads é de 10 a 100 vezes mais rápido que criar processos inteiros.

  • O que COMPARTILHAM: Variáveis globais, código, arquivos abertos e espaço de endereçamento (Memória).
  • O que NÃO compartilham (Cada thread tem o seu): Pilha (Stack), Registradores e Contador de Programa (PC).
  • User vs Kernel: Threads de nível de usuário são invisíveis para o SO e gerenciadas por bibliotecas (mudança de contexto não requer System Call). Threads de nível de Kernel são gerenciadas diretamente pelo SO (ex: Windows).

Sincronização e Condição de Corrida (Race Condition)

Race Condition: Acontece quando várias threads acessam e tentam modificar os mesmos dados ao mesmo tempo, gerando um resultado corrompido que depende de "quem chegou primeiro".

Região Crítica (Seção Crítica): É o trecho exato de código onde os dados compartilhados são manipulados. Só 1 thread pode estar nela por vez (Exclusão Mútua).

Espera Ocupada (Busy Waiting): É quando a thread fica em um loop vazio (`while(true)`) gastando muita CPU só pra ver se a trava liberou. É terrível para a performance.

Mutex vs Semáforos vs Monitores

Ferramenta Característica Principal
Mutex Fechadura Binária (0 ou 1). Usado estritamente para exclusão mútua. A mesma thread que trancou DEVE ser a que vai destrancar.
Semáforo Contador Inteiro. Permite que "N" threads acessem um pool de recursos. Usa as primitivas atômicas P (Wait/Down) para bloquear e V (Signal/Up) para liberar. Pode ser alterado por threads diferentes.
Monitores Módulo de software de alto nível. O compilador garante que os métodos sejam de exclusão mútua automática. Muito mais fácil para o programador do que gerenciar Semáforos na mão.

Sistemas de Arquivos e Permissões

O Linux gerencia discos dividindo em partições lógicas e usa Inodes para rastrear as informações (metadados: tamanho, dono, ponteiros para blocos). O Inode NÃO guarda o nome do arquivo, quem guarda isso é a estrutura de "diretório".

Permissões Especiais:

  • SUID (Set User ID): Quando você executa o programa, você ganha temporariamente os poderes do DONO do arquivo. Símbolo 's' no lugar do 'x' do usuário (rwsr-xr-x).
  • Sticky Bit: Aplicado a diretórios (ex: /tmp). Faz com que apenas o DONO de um arquivo possa apagá-lo ali de dentro, mesmo que a pasta seja de escrita pública. Símbolo 't' no final (rwxrwxrwt).

Flashcards de Revisão (Toque para Virar)

O que faz o comando `usermod -aG sudo maria`?

Adiciona a usuária Maria ao grupo 'sudo' sem removê-la de seus grupos anteriores (graças ao -a de append).

Se eu executar `chmod 751 script.sh`, como ficam as permissões?

Usuário = 7 (RWX), Grupo = 5 (R-X), Outros = 1 (--X). O Dono faz tudo, Grupo lê/executa e Outros só executam.

No Semáforo, o que acontece na operação P (Wait/Down) se o valor for ZERO?

A thread é bloqueada (vai dormir) na fila do semáforo, até que alguma outra thread execute a operação V (Signal/Up) incrementando o valor.

Como colocar um processo longo para rodar em 2º plano que continue mesmo fechando o terminal?

Usando `nohup [comando] &`. O `&` joga para o background, e o `nohup` (No Hang Up) impede a morte via logout.

Threads vs Processos: O que eles NUNCA compartilham entre si?

Pilha (Stack), Registradores de CPU e Contador de Programas (PC).

Qual a diferença fundamental entre um Hard Link e um Link Simbólico (Soft Link)?

Hard Link aponta pro Inode direto (não pode cruzar disco). Link Simbólico (`ln -s`) é um atalho de texto que aponta pro "caminho" e quebra se o original for deletado.
